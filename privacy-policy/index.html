Executive Summary & Data Sovereignty

At OmniIDE, we believe that software source code represents the core intellectual asset of every software engineer, technology startup, and enterprise. Developers must maintain uncompromised, absolute sovereignty over their local codebases, AI prompts, execution logs, and development workflows.

Unlike conventional cloud-bound developer tools that transmit private code snippets, file paths, or developer telemetry to remote server infrastructure, OmniIDE is engineered from the ground up as a Local-Native Integrated Development Environment (IDE). All file parsing, self-healing terminal loops, and syntax analyses operate 100% locally inside your hardware environment.

The Golden Privacy Rule of OmniIDE

We do NOT track your keystrokes. We do NOT collect your source code. We do NOT train AI models on your private intellectual property. We do NOT maintain remote tracking servers that monitor your IDE sessions.

This policy serves as an explicit, legally binding disclosure of how data is handled across our desktop software, official extensions, and online developer portals.

02

Scope & Applicability

This Privacy Policy applies strictly and exclusively to:

  • The official OmniIDE Desktop Application binaries (Windows 10, Windows 11, macOS, and Linux releases).
  • The primary web portal located at https://omniide.com and all subpages (including /docs/, /blog/, and /omni/).
  • Official open-source extensions, CLI tools, and plugins distributed under the OMNI-IDE-in GitHub organizational repository.

This policy does not govern third-party VS Code extensions that you choose to manually install from external marketplaces, nor does it govern third-party cloud AI APIs configured independently by the user.

Voluntary Information Disclosures

OmniIDE only processes personal information when a user voluntarily initiates contact through our official public communication channels:

  • Website Contact Forms: When submitting an inquiry via our site contact form (handled via Web3Forms), we receive your name, email address, inquiry topic, and message content solely to respond to your request.
  • GitHub Contributions: When submitting issues, feature requests, or pull requests on our GitHub organization (github.com/OMNI-IDE-in), your public GitHub profile name and comments are subject to GitHub's Privacy Statement.
05

AI Model Integrations & Cloud Data Flows

OmniIDE features a hybrid AI architecture supporting both local offline models and cloud AI APIs. You maintain 100% control over which execution mode is active:

A. Local Engine (Ollama Integration):
When using local model execution via Ollama, all inference runs 100% offline on your device’s GPU/CPU. Zero internet connections or data packets leave your hardware during local model interactions.

B. Cloud Provider Models (Google Gemini API):
When configuring OmniIDE to connect to Google Gemini API via a "Bring Your Own Key" (BYOK) model, your requests travel directly from your machine to Google’s official API endpoints (generativelanguage.googleapis.com). OmniIDE servers do not sit in the middle, record, or inspect these API payloads. Your data handling with cloud models is governed directly by Google API's Data Privacy Terms.

06

Hardened Local Sandbox & Process Security

To ensure security when running autonomous agentic workflows (such as Omni-Agent terminal self-healing), OmniIDE executes agent commands inside a Hardened Local Sandbox.

This local container boundary prevents agent scripts from modifying system configuration files outside your designated project workspace, protecting operating system stability and user security.

07

Cookies & Web Tracking Transparency

Our website (https://omniide.com) utilizes basic Google Analytics (gtag.js) solely to inspect aggregate visitor statistics (such as overall page views and browser device distribution).

No code snippets, personal identifiers, or IDE usage data are ever linked with web analytics cookies.

08

Storage Controls & Complete Erasure Rights

Because all workspace settings, local model weights, and chat logs are stored strictly on your local disk (under your OS user directory ~/.omniide or %APPDATA%\OmniIDE), you possess absolute authority over data retention and erasure.

Deleting your local project directory or clearing the application data directory permanently erases all stored state from your machine instantly.

09

Global Regulatory Compliance

OmniIDE aligns with international data privacy standards:

  • India Digital Personal Data Protection Act 2023 (DPDP): Fully compliant with strict data minimization and purpose specification rules.
  • EU General Data Protection Regulation (GDPR): Compliant with Article 5 principles of privacy by design and default.
  • California Consumer Privacy Act (CCPA/CPRA): We do not sell, rent, or trade any personal information.
